Clicking “Denied Customers” showed the whole dashboard in the section
• Why: The controller sometimes returned a redirect for AJAX. Browsers don’t auto-follow server redirects inside $.ajax; you just get the redirected page’s HTML injected.
• Fix: Never redirect for these AJAX list calls. Always return the partial with collect() as the default: $grouped->get($status, collect()). Then the partial renders (empty or not).

Duplicate id="placeholderSection" inside the partial
• Why: The inner partial reused the same collapse target id as the dashboard, so Bootstrap’s collapse toggles acted on the wrong element.
• Fix: Remove the id in the partial (or make it unique). The parent keeps #placeholderSection; the partial just renders content.

Right-click delete menu was inconsistent after AJAX loads
• Why: Browser’s default context menu wasn’t always suppressed, and handlers bound before .html() replacement don’t apply to new nodes.
• Fix: Use a delegated contextmenu listener on document (or container) and call event.preventDefault() to suppress the native menu; keep handlers outside the replaced fragment so they continue to work after AJAX.
